    4. TP 1: Basisbeheer van een loonstrook

    4.1. Introduction

    Om het eerder geleerde in de praktijk te brengen, stellen we nu een opdracht voor waarbij je een Android-client voor tablets moet schrijven waarmee de loonberekeningen van de medewerkers van een vereniging kunnen worden gesimuleerd.

    De applicatie krijgt een client-serverarchitectuur:

    • de server [1] wordt meegeleverd;
    • de Android-client [2] moet worden gebouwd.

    4.2. De database

    4.2.1. Definitie

    De statische gegevens die nodig zijn voor het opstellen van de loonstrook worden opgeslagen in een database die we hierna de dbpam- e noemen. Deze database bevat de volgende tabellen:

    Tabel EMPLOYES: bevat informatie over de verschillende kinderopvangmedewerkers

    Structuur:

    ID
    primaire sleutel
    VERSION
    versienummer – loopt op bij elke wijziging van de rij
    SS
    sofinummer van de werknemer – uniek
    NOM
    naam van de werknemer
    PRENOM
    zijn voornaam
    ADRESSE
    zijn adres
    VILLE
    zijn woonplaats
    CODEPOSTAL
    zijn postcode
    INDEMNITE_ID
    vreemde sleutel in het veld [ID] van de tabel [INDEMNITES]

    De inhoud zou als volgt kunnen zijn:

    Tabel COTISATIONS: bevat de percentages die nodig zijn voor de berekening van de sociale premies

    Structuur:

    ID
    primaire sleutel
    VERSION
    versienummer – loopt op bij elke wijziging van de regel
    CSGRDS
    percentage: algemene sociale bijdrage + bijdrage aan de aflossing van de sociale schuld
    CSGD
    percentage: aftrekbare algemene sociale bijdrage
    SECU
    percentage: sociale zekerheid, weduwschap, ouderdom
    RETRAITE
    percentage: aanvullend pensioen + werkloosheidsverzekering

    De inhoud zou als volgt kunnen zijn:

    De sociale premiepercentages zijn onafhankelijk van de werknemer. De bovenstaande tabel bevat slechts één rij.

    Tabel INDEMNITES: bevat de gegevens die nodig zijn voor de berekening van het te betalen loon.
    ID
    primaire sleutel
    VERSION
    versienummer – loopt op bij elke wijziging van de regel
    INDICE
    verwerkingsindex – uniek
    BASEHEURE
    nettoprijs in euro's voor een uur wachtdienst
    ENTRETIENJOUR
    vergoeding voor levensonderhoud in euro per dag dienst
    REPASJOUR
    maaltijdvergoeding in euro per dag dienst
    INDEMNITESCP
    vergoeding voor betaald verlof. Dit is een percentage dat op het basisloon moet worden toegepast.

    De inhoud zou als volgt kunnen zijn:

    Er zij op gewezen dat de vergoedingen van kinderoppas tot kinderoppas kunnen verschillen. Ze zijn namelijk gekoppeld aan een specifieke kinderoppas via diens salarisindex. Zo heeft mevrouw Marie Jouveinal, die een salarisindex van 2 heeft (tabel EMPLOYES), een uurloon van 2,1 euro (tabel INDEMNITES).

    4.3. Installatie van de webserver / jSON

    4.3.1. Installatie

    Het Java-uitvoeringsbestand van de webserver / jSON wordt u verstrekt:

     

    Ga als volgt te werk om de webserver / jSON te starten:

    • start SGBD MySQL;
    • controleer of BD en [dbpam_hibernate] aanwezig zijn;
    • open een venster DOS;
    • ga naar de map van het jar-bestand;
    • typ de opdracht: 
    java -jar pam-server-01-all-1.0.jar

    Hierbij wordt ervan uitgegaan dat het binaire bestand [java.exe] zich in de map PATH op uw computer bevindt. Als dit niet het geval is, voer dan het volledige pad naar [java.exe] in, bijvoorbeeld:

    D:\Programs\devjava\java\jdk1.8\bin\java -jar pam-server-01-all-1.0.jar

    4.4. Tests van de Android-client

    Hieronder vindt u het uitvoerbare binaire bestand van de voltooide Android-client:

      

    Sleep het bovenstaande binaire bestand [pam-client.apk] met de muis naar een tabletemulator [GenyMotion]. Het wordt dan opgeslagen en vervolgens uitgevoerd. Start ook de webserver / jSON als u dat nog niet hebt gedaan. De Android-client heeft als doel de door de webserver / jSON teruggestuurde informatie op te halen en op te maken. De verschillende schermen van de Android-client zijn als volgt:

    U moet eerst verbinding maken met de webservice / jSON:

    • in [1], geef je de URL van de webservice / jSON op. Stel in de emulator een van de adressen IP of PC in (maar niet 127.0.0.1). Voer op een tablet het wifi-adres van de webserver in / jSON en schakel de firewall van de server uit, indien aanwezig, omdat deze inkomende verbindingen zou kunnen blokkeren;
    • in [2] log je in;

    Je komt dan op de simulatiepagina terecht:

    • in [3] kiest men een medewerker;
    • in [4] voer je een aantal uren in;
    • op [5] voer je het aantal dagen in;
    • in [6] wordt de simulatie opgestart;

    De resulterende simulatiepagina ziet er als volgt uit:

    • in [7], de verkregen simulatie;
    • in [8] wordt deze opgeslagen;

    • in [9], de lijst met simulaties;
    • in [10], wordt een simulatie verwijderd;

    • in [11], er zijn geen simulaties meer;
    • in [12], keert men terug naar het simulatieformulier;

    • in [13] verschijnt het formulier weer;
    • in [14] keert men terug naar de configuratiepagina;

    • met [15] komt u terug bij het oorspronkelijke inlogformulier.

    4.5. Te doen

    Het eerder gepresenteerde skelet van de Android-client wordt u aangereikt. Het is opgebouwd op basis van het project [client-android-skel] dat in paragraaf 2 wordt beschreven.

      

    Het project is uitvoerbaar en bevat al de benodigde weergaven. Er hoeft alleen nog wat code te worden toegevoegd om de applicatie te laten doen wat ze moet doen. Ga als volgt te werk:

    • voer de volledige versie uit om inzicht te krijgen in het werk dat moet worden gedaan;
    • voer de vereenvoudigde versie uit en bestudeer de code daarvan. Deze volgt de ontwerpmethoden die op de voorgaande pagina's zijn gebruikt;
    • voeg de ontbrekende code toe;
